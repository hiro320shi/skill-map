## 2. エンジニアに必要なスキル項目（バックエンドWeb想定）

ざっくりカテゴリ分けするとこんな感じです。

### A. コンピュータ・プログラミング基礎

- コンピュータサイエンス基礎
  - データ構造（配列・リスト・スタック・キュー・ハッシュ・ツリー など）
  - アルゴリズム（探索・ソート・計算量の感覚）
- OS・コマンドライン
  - Linux 基本コマンド（ファイル操作、プロセス確認、権限など）
  - Windows の基本操作（PowerShell, cmd の違い など）
- ネットワーク基礎
  - TCP/IP, HTTP, DNS, ポートの概念
  - REST API の考え方
- セキュリティ基礎
  - XSS, CSRF, SQLインジェクション などの基本的な脆弱性
  - 認証・認可（Basic認証, セッション, JWT など）

### B. 言語・フレームワーク

- メイン言語（例：Java / C#）
  - 基本文法（制御構文・クラス・例外処理）
  - コレクション・ジェネリクス
  - オブジェクト指向（クラス設計・継承・インターフェース）
- フレームワーク
  - Javaなら：Spring Boot
  - C#なら：ASP.NET Core
- スクリプト/補助言語
  - JavaScript（簡単なフロント修正ができる程度）
  - Shell（bash）や PowerShell（簡単な自動化）

### C. Web・フロント基礎

- Web 基本
  - HTTPメソッド（GET/POST/PUT/DELETE）
  - ステータスコード（200/400/404/500 など）
- フロント基礎
  - HTML / CSS の基礎
  - JavaScript の基礎
  - Ajax / Fetch / JSON の扱い

「フロントガッツリ」は後回しでも、**APIと連携できるくらいの理解**があると強いです。

### D. DB・データベース

- RDB（リレーショナルデータベース）
  - MySQL / PostgreSQL / SQL Server など
  - SQL（SELECT / INSERT / UPDATE / DELETE / JOIN）
  - トランザクション・インデックス・実行計画の基礎
- データモデリング
  - 正規化、ER 図、テーブル設計
- ORマッパー
  - Java: JPA / Hibernate
  - C#: Entity Framework

### E. インフラ・クラウド・コンテナ

- OS
  - Linux サーバの基本操作
- コンテナ
  - Docker / Docker Compose
- クラウド
  - AWS 基本サービス
    - EC2 / S3 / RDS / Lambda / CloudWatch など
- ネットワーク / アーキテクチャ
  - VPC / サブネット / セキュリティグループ
  - 負荷分散・スケールアウトのイメージ

### F. 開発プロセス・ツール

- バージョン管理
  - Git（clone, branch, commit, push, pull, merge, rebase）
  - GitHub / GitLab の使い方（PR/MR, コードレビュー）
- テスト
  - 単体テスト（JUnit, NUnit など）
  - モック、DIの考え方
  - 結合テスト・APIテスト（Postman, Swagger UI など）
- CI/CD
  - GitHub Actions / GitLab CI（ビルド・テスト自動化のイメージ）
- タスク管理
  - Issue管理（Backlog, Jira, GitHub Issues）
  - チケット駆動の考え方

### G. 設計・アーキテクチャ

- オブジェクト指向設計
  - SOLID原則
  - レイヤードアーキテクチャ（Controller/Service/Repository など）
- アーキテクチャパターン
  - MVC / MVVM / クリーンアーキテクチャのざっくりした理解
- 設計ドキュメント
  - シーケンス図 / クラス図 / ER図 など

### H. ソフトスキル・その他

- 要件定義・ヒアリング
- コミュニケーション（報告・相談・レビューのやりとり）
- ドキュメンテーション
  - Markdown での設計書、README、手順書作成
- 学習スキル
  - ドキュメントや公式リファレンスを読む力
  - 小さく試しながら学ぶ習慣
